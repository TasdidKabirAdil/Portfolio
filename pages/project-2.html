Azure DevOps CI/CD Pipeline Implementation

Overview

As part of a DevOps Implementation course, I built a deployment-ready version of a Health Monitor web app using ASP.NET MVC and managed the entire lifecycle through Azure DevOps. The project followed Agile practices with Azure Boards tracking epics, features, user stories, and tasks. Source control was handled using both GitHub and Azure Repos, with linked branches and pull requests. CI pipelines were configured in Azure DevOps to automatically build and test code on every push, integrating static analysis using Roslyn Analyzer. CD pipelines deployed the app to an IIS server using Web Deploy, completing a secure and automated release process. The project demonstrates end-to-end DevOps practices including test coverage, artifact generation, and release automation.

Key Features

Agile Project Management

The project was structured around Agile principles using Azure Boards to manage all development work across epics, features, user stories, and tasks. Each item was linked to branches, commits, and pull requests, ensuring traceability and clear development flow. This allowed for consistent progress tracking and real-time collaboration within the team.

Continuous Integration (CI)

A robust CI pipeline was created in Azure DevOps, triggered automatically on GitHub pushes. The pipeline included build validation, static code analysis using Roslyn Analyzer, and automated testing with code coverage reports. This ensured that all code changes were verified early and met quality standards before integration.

Continuous Deployment (CD)

The project implemented a secure CD pipeline using Azure DevOps Release pipelines. Artifacts from the CI process were deployed directly to an IIS Web Server using Web Deploy. This end-to-end automation enabled faster, consistent releases while maintaining deployment integrity through a connected GitHub PAT-based authentication system.
